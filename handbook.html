The NOVA Handbook

A general, target-independent guide to reproducing NOVA's processing pipeline by hand. Per-run recipe pages record exact choices for one image; this page explains the complete toolbox in pipeline order.

PixInsight

A specialist commercial workbench. Check the vendor for current license pricing and terms. Some XTerminator tools are separate purchases.

Siril

Free, open-source astronomical image processing with strong calibration, stacking, and scripting.

SASpro

Seti Astro Suite Pro is currently a free download with a suggested donation; it bundles stacking, background, AI cleanup, wave-scale, and finishing tools.

Measure → try → measure → adjust. Treat every recipe as a starting hypothesis. Measure, make the smallest useful change, measure again, and back off when the pixels show damage. Interfaces evolve: “verify” marks intentionally untested paths or parameters.

Acquisition & stacking

Turning a night of raw sub-frames into one calibrated linear master.

Subframe Culling

Ranks every individual sub-frame and rejects the worst before they ever reach the stack. A single trailed, cloud-hazed or wind-shaken frame can drag down an otherwise good integration.

Process: SubframeSelector
  1. Add all your light frames and run Measure.
  2. In the plots, sort by FWHM and Eccentricity; set an approval expression such as FWHM < 2.8 && Eccentricity < 0.55 && SNRWeight > 0.6 (use the medians as a guide — reject roughly the worst 5–10%).
  3. Set Output → Action: Output approved only and run; the approved subs go forward to registration.
It also writes per-frame weights you can feed to ImageIntegration.
Plot tab / frame selector.
  1. After star registration, compare FWHM, weighted FWHM, roundness, background, and star count in the Plot tab (F5).
  2. Exclude obvious outliers directly from the plot or frame selector, initially only the worst 5–10%, then re-stack and compare sharpness against the lost √N signal.
Frame evaluation before integration. Use SASpro's current subframe/stacking quality view, if available, to reject FWHM, eccentricity, star-count, or transparency outliers; otherwise cull in Siril or PixInsight first. Verify the current module and metric labels.

Stacking

Calibrates, registers (aligns) and integrates all surviving sub-frames into one master FITS. This is where read-noise is beaten down and the faint signal first becomes visible.

Script: WeightedBatchPreprocessing (WBPP) does the whole chain in one go.
Manual equivalent, in order:
  1. ImageCalibration — apply master dark / flat / bias.
  2. CosmeticCorrection — hot/cold pixel map.
  3. Debayer — RGGB for the SeeStar OSC sensor.
  4. StarAlignment — register all frames to a reference (enable Generate drizzle data if you'll drizzle).
  5. ImageIntegration — Linear Fit clipping or Winsorized Sigma, noise-evaluation weighting, additive+scale normalization.
  6. (optional) DrizzleIntegration at 2× for extra resolution.
Scripts → OSC_Preprocessing (or the manual sequence).
  1. Convert lights to a sequence, calibrate (calibrate with your master flat/dark/bias), then register (Global Star Alignment).
  2. stack rej 3 3 -norm=addscale -output_norm — Winsorized sigma 3/3, additive+scaling normalization.
  3. For 2× drizzle add -drizzle to the register step.
Image MM (Multi-Method) integration.
  1. Load calibrated, registered frames; Image MM weights and rejects, producing a detail-forward stack. (SASpro doesn't register — stack in Siril/PI first, or feed pre-registered frames.)

Pre-process — linear cleanup

Cheap, lossless fixes applied to the master before any heavy processing.

Crop

Trims the ragged stacking borders — the low-coverage edges where dithering and field rotation mean only some frames contributed, so the signal there is noisier and the gradient is worst.

Process: DynamicCrop (or the Crop instance).
  1. Open DynamicCrop on the integration and drag the rectangle inside the ragged low-coverage border.
  2. To find the true common-coverage area, integrate with Generate integration weight map and crop to where the weight is full.
  3. Keep north-up; DynamicCrop updates the WCS/CRPIX for you.
Selection + Crop.
  1. Drag a rectangle inside the ragged stacking border, then Image Processing → Crop (or the crop x y w h command).
  2. Siril preserves the WCS after the crop, so plate-solve stays valid.
Not a SASpro step. Crop in Siril or PixInsight before you bring the stack into SASpro for stretching.

Remove Pedestal

Removes the ADC bias pedestal — the constant electronic offset the camera adds so the sensor never reads true zero. It sets the real black point to exactly zero.

Process: PixelMath (or set it at calibration time).
  1. The clean way is ImageCalibration → Output pedestal when you build the master, so it's already removed.
  2. On an existing master, use PixelMath per channel: $T - min($T) (rescale off), which pins true black to zero while keeping channel ratios intact.
PixelMath / offset.
  1. Set an output pedestal at calibration (-pedestal=) so it's gone before stacking, or
  2. subtract the floor afterward with PixelMath: iif(I>m, I-m, 0) where m is the background level from Statistics.
Not a SASpro step. Handle the pedestal at calibration (Siril/PI) — SASpro assumes a clean linear stack.

Cosmetic Correction

Removes residual hot and cold pixels — single bright/dark pixels left by sensor defects, cosmic-ray hits, or stacking misses that survived rejection.

Process: CosmeticCorrection.
  1. Run on the registered frames (ideally pre-integration).
  2. Without a master dark, enable Use Auto detect and set Hot/Cold Sigma ≈ 3–5 — only true outliers, never star cores.
  3. Use Real-Time Preview to confirm hot pixels vanish while stars are untouched.
Cosmetic correction (at calibration).
  1. In Calibration enable Cosmetic correction with CC from sigma: Cold ≈ 3, Hot ≈ 3–5.
  2. Or run find_hot to build a bad-pixel map and apply it. Confirm star cores are untouched in the preview.
Not a SASpro step. Cosmetic/hot-pixel repair belongs at calibration in Siril or PixInsight.

Linear processing

The science-critical stage. Everything here happens on linear data, where pixel values are still proportional to photons — the only place gradient removal, colour calibration and deconvolution are mathematically valid.

Background Extraction

Removes large-scale gradients — light pollution, moon glow, vignetting, and amp glow — that tilt or dome the background. The single most important step for a clean, neutral sky.

Process: GradientCorrection (modern) or DynamicBackgroundExtraction (DBE).
  1. DBE: place samples only on star-free background; raise Tolerance so samples on faint signal are rejected.
  2. Set Correction: Subtraction for additive light pollution, Division for vignetting.
  3. Inspect the generated background model — it must be smooth, with no imprint of the galaxy/nebula (that means it's eating real signal).
Do this on linear data only.
Image Processing → Background Extraction.
  1. Samples per line ≈ 20; raise Tolerance so samples on nebulosity are rejected. Use RBF for busy gradients, polynomial (deg 4) for smooth ones.
  2. Correction: Subtraction for light pollution. Run on linear data.
  3. Check the generated model shows no imprint of the target.
ABE — Automatic Background Extraction.
  1. Run ABE on the linear stack; it fits a polynomial + RBF model and subtracts it. Increase the sample count on busy fields.
  2. This is the SASpro tool the pipeline uses for gradient removal.

Color Calibration

Photometric colour calibration (SPCC). Sets a physically correct white balance so star colours — and therefore nebula/galaxy colours — match their real spectral types instead of the camera's raw response.

Process: SpectrophotometricColorCalibration (SPCC).
  1. Plate-solve first (ImageSolver / Image > Astrometry) so SPCC knows the field — it needs the local Gaia DR3 database configured.
  2. Pick the Sony Color Sensor + filter that matches the SeeStar S50; white reference Average Spiral Galaxy is a safe default.
  3. Run on linear data; check the residual scatter plot for a tight fit.
If the solve fails, SPCC can't run — fall back to BackgroundNeutralization + manual ColorCalibration.
Image Processing → Spectrophotometric Color Calibration (SPCC) (Siril 1.2+; else Photometric CC).
  1. Plate-solve first (Image Information → Astrometry).
  2. Choose the OSC sensor + your filter; run on linear data.
  3. No solve? Fall back to Color Calibration → Background Neutralization + manual white reference.
SASpro has no SPCC. Use its White Balance / Background Neutralization for a quick balance, or do photometric color calibration in PixInsight/Siril, then return to SASpro for stretching.

Deconvolution

Recovers spatial detail that seeing and optics blurred away — tightens stars and sharpens fine structure. Done on linear data, where the blur is a true convolution with the PSF.

Process: BlurXTerminator (the modern choice).
  1. Run on linear data, after BGE/SPCC.
  2. Set Automatic PSF; start Sharpen Stars ≈ 0.25 and Sharpen Nonstellar ≈ 0.7–0.9.
  3. Classical alternative: Deconvolution with an external PSF from DynamicPSF and a star mask + local deringing.
Image Processing → Deconvolution.
  1. Generate a PSF from stars (PSF from stars), then Richardson–Lucy, ~10–20 iterations, with a moderate regularization to avoid ringing.
  2. Linear data only. Siril has no AI decon — for BlurXTerminator-class results use PixInsight or SASpro's Cosmic Clarity Sharpen instead.
Cosmic Clarity — Sharpen.
  1. Run the Cosmic Clarity Sharpen module (Stellar / Non-Stellar modes), amount ~0.5 stellar / 0.3 non-stellar — the AI-decon equivalent of BlurXTerminator.
  2. Linear data; check star profiles for ringing.

Denoise Linear

Reduces noise while the data is still linear — the most effective place to denoise, because the noise statistics are well-behaved and the stretch hasn't yet amplified them.

Process: NoiseXTerminator.
  1. Run on linear data (it's linear-aware).
  2. Denoise ≈ 0.8, Detail ≈ 0.15 — back off Detail if structure smears.
  3. Classical alternative: MultiscaleLinearTransform with noise reduction on the first 3–4 wavelet layers, or TGVDenoise.
Image Processing → Denoise.
  1. Anscombe VST + NL-Bayes; leave modulation at 1.0, enable cosmetic correction off. Run on linear data.
  2. Preview at 1:1 to confirm faint structure survives.
Cosmic Clarity — Denoise.
  1. Run on linear data, strength ~0.5; it's a learned denoiser that preserves faint structure better than a blur.

Star Sharpen

BXT in correct-only mode: fixes star shape (roundness, elongation) without deblurring the background. Runs after denoise on dense fields where denoise has bloated the stars.

Process: BlurXTerminator (Correct Only).
  1. Enable Correct Only — this fixes star shape with no deblurring.
  2. Set both Sharpen amounts to 0; let it round and de-elongate stars only.
  3. Run after denoise if denoise has bloated the stars on a dense field.
No direct equivalent. Siril sharpens stars only via the same deconvolution pass. For a stars-only correct-only sharpen, use PixInsight BlurXTerminator (Correct Only) or SASpro Cosmic Clarity on the star layer.
Cosmic Clarity — Sharpen (Stellar only) on the star layer, or skip in SASpro and let PixInsight BlurXTerminator Correct-Only handle stars.

Star separation & stretch

The perceptual turning point: stars are split off, the faint signal is stretched from linear to non-linear, and the stars are stretched separately.

Remove Stars Linear

Splits the image into a starless layer and a stars-only layer before stretching. The starless layer goes through the rest of the pipeline; the stars are stretched separately and screened back in at the end.

Process: StarXTerminator.
  1. Enable Generate star image so you get both a starless layer and a stars-only layer that recombine by screen.
  2. Run it before stretching so the nebula can be pushed without bloating stars.
  3. Check the starless layer for dark pits where bright stars were.
Image Processing → StarNet Star Removal (StarNet++ must be configured in Preferences → Miscellaneous).
  1. Run on the linear image; Siril produces a starless image and a stars-only image you can recombine later.
Cosmic Clarity — DarkStar (or StarNet).
  1. DarkStar removes stars and can generate the stars-only layer for later recombination — the SASpro path the pipeline uses.

Stretch

The critical perceptual step: converts linear data (where the faint signal is a fraction of a percent above black) into a non-linear image the eye can read. Everything before this is invisible; this is where the picture appears.

Process: GeneralizedHyperbolicStretch (GHS), HistogramTransformation, or MaskedStretch / ArcsinhStretch.
  1. Read the auto-STF first (ScreenTransferFunction → wrench) to see a good starting black/midtone point.
  2. GHS: set SP (symmetry point) near the sky peak in the log histogram, raise D for strength, b for local contrast — preview live.
  3. Or drag STF into HistogramTransformation and apply to bake it in. Keep RGB linked to protect the SPCC colour.
  4. Aim the sky background into its per-type band and leave highlight headroom (don't clip the core).
Image Processing → Generalized Hyperbolic Stretch (GHS) for control, or Histogram Transformation / Asinh for a quick stretch.
  1. GHS: set the Symmetry point (SP) just above the background, raise D (stretch intensity) and b (local intensity) to lift the galaxy without blowing the core.
  2. Autostretch (the eyeball icon) is a fast starting point but not final.
Statistical Stretch.
  1. Set the target median (background) and enable linked channels to protect color; it places the sky and lifts signal in one measured step — the SASpro tool behind the pipeline's stretch candidates.
  2. For galaxies, keep the target median low (darker sky) and let contrast come from Curves after.

Stretch Stars

Stretches the stars-only layer on its own, so star brightness, size and colour are controlled independently of the nebula.

Process: ArcsinhStretch or HistogramTransformation on the stars-only layer.
  1. Stretch the star image on its own, gentler than the nebula — keep stars secondary (p90 ≈ 0.25).
  2. ArcsinhStretch preserves star colour well; raise Stretch factor gradually with Real-Time Preview.
Stretch the stars-only image separately with GHS/Asinh, lighter than the galaxy, then bump saturation ~1.2 and SCNR the green before recombining.
Star Stretch.
  1. On the stars-only layer: the Star Stretch module auto-picks a stretch from the star profile, with a saturation boost and SCNR — exactly what the pipeline runs.

Non-linear — colour & detail

Aesthetic and structural shaping on the stretched starless image.

Scnr

Selective Colour Noise Reduction — removes the green/teal cast that OSC (one-shot-colour) sensors like the SeeStar produce, where the green channel is over-represented.

Process: SCNR.
  1. Colour to remove: Green; protection method Average Neutral, amount 1.0.
  2. Enable Preserve lightness.
  3. Skip on narrowband/duo-band — it would strip real OIII that lands in green/teal.
Remove Green Noise (SCNR).
  1. Apply a modest amount to a stretched broadband image and compare neutral background samples before/after.
  2. Skip or mask it for narrowband/duo-band data where green/teal can be real OIII.
Verify the amount against the current preview; there is no universal safe value.
Remove Green / SCNR-style correction. Suppress only enough green to neutralize broadband background samples; skip it for narrowband data carrying real OIII. Verify the current control name, location, and scale.

Narrowband Norm

PixInsight NarrowbandNormalization — balances the Ha and OIII channels of duo-band data so neither dominates unnaturally.

Script: NarrowbandNormalization (Script → Utilities).
  1. Feed the Ha and OIII channels; choose Equalize to balance them or a max-stars method to keep a dominant Ha.
  2. It's an aesthetic balance — judge against your intended palette, not a metric.
RGB Composition / Pixel Math.
  1. Load the aligned Ha/OIII/SII masters in RGB Composition and use Adjust layers brightness, or measure comparable regions and scale the images explicitly in Pixel Math.
  2. Re-measure before compositing; preserve the intended dominant channel rather than forcing equal brightness.
Narrowband Normalization. Load the aligned HOO/SHO/HSO/HOS channels, measure the output against the inputs, and avoid normalizing away the intended dominant channel. The installed SASpro 1.18.0 confirms the module name; verify its controls and output mode in the current UI.

Background Neutralize

Post-stretch background neutralisation — removes any residual colour cast left in the darkest background regions after stretching.

Process: BackgroundNeutralization.
  1. Draw a preview over a genuinely empty sky region and set it as the Reference image.
  2. Run to force R≈G≈B in the darkest background after stretch.
Image Processing → Color Calibration → Background Neutralization, then Remove green noise (SCNR) at amount ~0.5 to kill the green cast.
Background Neutralization / Remove Green. Neutralize the sky, then SCNR the green cast — do this before the color boost.

Color Boost

Hue-selective saturation boost — lifts the specific colours that matter for the target type while leaving the rest alone.

Process: ColorSaturation (hue-selective) or a saturation CurvesTransformation.
  1. ColorSaturation: lift the curve only over the target hues — blue (spiral arms) or red/cyan (Ha/OIII) — and hold the rest flat.
  2. Protect stars/background with a range or star mask so the boost doesn't leak into the sky.
Image Processing → Color Saturation.
  1. Raise saturation modestly (background protection on) so only the galaxy midtones gain color, not the sky.
Color / Saturation controls (or the SCNR + saturation pass). Boost galaxy midtone saturation with background protection so the sky stays neutral.

Clahe

Contrast Limited Adaptive Histogram Equalisation — local contrast enhancement that brings out fine structure the global stretch left flat.

Process: LocalHistogramEqualization (LHE).
  1. Set Kernel Radius to the structure scale you want to pop (≈64–110), Contrast Limit ≈ 1.5–2.5, low Amount to start.
  2. Use a luminance/structure mask so flat background isn't roughened.
CLAHE local contrast enhancement.
  1. Work on the stretched starless image; Siril recommends CLAHE for non-linear data. Start from the preview defaults and keep the clip limit conservative.
  2. Compare at 1:1 and back off if noise or tile boundaries appear.
Command: clahe cliplimit tileSize.
CLAHE. On the stretched starless layer, use conservative tile-grid and clip-limit settings; back off if tiles, noise, or crunchy edges appear. The installed SASpro 1.18.0 confirms the module name; verify parameter scale in the preview.

Noise Reduction

A second, post-stretch noise pass for residual grain the stretch amplified that linear denoise couldn't pre-empt.

Process: NoiseXTerminator (again, gentler) on the stretched image.
  1. The stretch changed the noise statistics, so this is a light touch-up — small Denoise, high Detail.
  2. Mask to background-only if you only want to clean the sky.
Post-stretch denoise.
  1. Use Siril's current denoise tool lightly, preferably through a background mask.
  2. At 1:1, grain should fall without erasing faint dust or small stars.
Verify the current module name and controls; do not reuse linear-data strengths blindly.
Cosmic Clarity — Denoise, light nonlinear pass. Use less strength than the linear cleanup, protect real small-scale structure, and compare at 1:1. Verify current Stellar/Non-Stellar options and strength scale.

Curves

Parametric tone curve for the final brightness/contrast/colour shaping, applied to the starless stretched image before recombination.

Process: CurvesTransformation.
  1. Gentle S-curve on the RGB/K channel for contrast; pull a highlight rolloff point down to protect a bright core.
  2. Keep the shadow end anchored — don't lift the already-dark sky (that kills contrast).
  3. Use the Saturation channel here too for subtle colour depth.
Image Processing → Curves (Siril 1.2+).
  1. Gentle S-curve: lift the upper-mid for contrast, pin the shadow point so the sky doesn't crush. Small moves only.
Curves Utility. Gentle S-curve for contrast; pin the shadow so the sky doesn't crush, lift the upper-mid where the galaxy structure lives.

Hdr Compression

Wavelet HDR compression — tames blown highlights (galaxy cores, bright nebula knots) while lifting faint structure, expanding the usable dynamic range.

Process: HDRMultiscaleTransform (HDRMT).
  1. Number of layers ≈ 6, Overdrive 0 to start; apply through a luminance mask that exposes only the blown core.
  2. Follow with a slight CurvesTransformation to recover global contrast HDRMT flattens.
Compress the bright core with masks and curves.
  1. Build a luminance/range mask exposing only the core.
  2. Use Curves or GHS locally to lower it while preserving the outer object, then restore slight global contrast.
No verified one-click HDRMT equivalent is recorded; verify current scripts before naming one.
WaveScale HDR. Start with conservative compression and mask-gamma settings; its built-in luminance mask protects darker structure while compressing highlights. Restore global contrast afterward. The installed SASpro 1.18.0 confirms the module and controls; verify their scale in the preview.

Dark Enhance

Wavelet dark-structure enhancement — boosts faint shadow detail: integrated flux nebulae (IFN), dust lanes, outer galaxy arms.

Process: LocalHistogramEqualization or MultiscaleLinearTransform with a dark mask.
  1. Mask to the faint shadow regions (IFN, dust, outer arms) and boost large wavelet scales gently.
  2. Stop as soon as the background starts lifting — that's noise becoming fake nebulosity.
Masked shadow-detail enhancement.
  1. Mask faint dust, IFN, or outer arms while excluding empty sky.
  2. Use gentle GHS/Curves or local contrast, then re-measure corner noise.
Verify current mask and local-contrast paths.
WaveScale Dark Enhancer. Enhance faint dust or outer arms conservatively, then re-measure the background so noise is not promoted into false nebulosity. The installed SASpro 1.18.0 confirms the module name; verify its controls and mask in the preview.

Color Sat

A uniform post-curves saturation lift to recover the muted star colours that statistical stretches wash out.

Process: ColorSaturation (uniform).
  1. A small global lift to recover star colour a statistical stretch washed out (HS delta ≈ 0.25).
  2. Avoid on globulars — it tends to introduce a green cast there.
Color Saturation.
  1. Apply a small global saturation recovery after stretch, using the preview or ROI to protect background and bright stars.
  2. Blink before/after; stop if the sky or globular stars turn green.
Choose the amount from the image; there is no universal safe value.
Color / Saturation controls. Make a small global recovery after stretch while protecting sky and bright stars; avoid it when it introduces a green cast. Verify the current saturation scale.

Recombination & finish

Stars are screened back in and final star-halo cleanup is applied.

Combine Stars Screen

Screen-blends the separately-stretched stars layer back onto the processed starless image — the inverse of the star split, restoring the stars to a fully-processed nebula/galaxy.

Process: PixelMath (screen blend).
  1. With the processed starless and stretched stars images open, run PixelMath: 1 - (1 - starless) * (1 - stars) (the screen operator, the exact inverse of StarXT unscreen removal).
  2. Check bright stars for doubling/halos from imperfect removal.
PixelMath screen blend. With the stretched starless and stars-only images loaded: ~((~starless)*(~stars)) (screen), or use Image Processing → Pixel Math to add the stars back over the galaxy.
Image Combine — Screen blend.
  1. Combine the stretched starless galaxy and the stars-only layer with the Screen blend mode to lay the stars back over the galaxy without clipping.

Halo Suppression

Reduces the colour halos and bloat around bright stars — the violet/blue rings refraction and the optics leave.

No native process — use a star mask + repair.
  1. Build a mask isolating bright-star halos (StarMask, or StarXT residual).
  2. Through the mask, pull the violet/blue ring with CurvesTransformation (reduce saturation + slight darken), or use MorphologicalTransformation erosion.
  3. On dense fields keep it minimal so you don't crush thousands of small stars or cast the field blue.
Mask and repair bright-star halos.
  1. Create a star/range mask isolating only colored rings.
  2. Through it, reduce saturation and slightly lower affected tones without changing star cores.
No verified direct halo tool is recorded; verify current scripts/plugins.
Halo-B-Gon. Isolate bright-star rings, reduce their saturation/brightness conservatively, and compare star cores before/after. The installed SASpro 1.18.0 confirms the module name; verify its detection and reduction controls in the preview.

Narrowband

Optional duo/tri-band palette compositing for emission targets.

Narrowband Composite

Combines separate mono Ha / OIII / SII channels into a false-colour RGB using a narrowband palette.

Process: PixelMath or ChannelCombination.
  1. SHO (Hubble): map R=SII, G=Ha, B=OIII via ChannelCombination.
  2. Foraxx-style: PixelMath blends, e.g. R = Ha*0.76 + OIII*0.24, G = OIII*0.85 + Ha*0.15, B = OIII.
  3. Then process the composite through the normal nonlinear chain.
RGB Composition / Pixel Math.
  1. Load aligned, normalized Ha/OIII/SII masters and assign colours in RGB Composition, or use three Pixel Math expressions for a custom SHO/HOO palette.
  2. Inspect every input and output channel for clipping before the non-linear chain.
Pixel Math variables must be loaded first; command-line image names use $name$.
Narrowband Integration (or Pixel Math for a custom formula). Load aligned, normalized Ha/OIII/SII channels, assign the desired SHO/HOO mix, and check every output channel for clipping. The installed SASpro 1.18.0 confirms both module names; verify the palette controls and formula ranges in the current UI.
